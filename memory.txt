ARCHITECTURE RULES (from SaaS-Architecture-v2.md)
--------------------------------------------------------------------------------

- Code identifiers: always "mate" (namespaces, project names, DI keys)
- UI/display text: always BrandInfo.BrandName (never hardcode "mate")
- Module project naming:
    mate.Modules.AgentConnector.*   (SINGULAR — no 's')
    mate.Modules.Auth.*             (SHORT — not "Authentication")
    mate.Modules.Testing.*
    mate.Modules.Monitoring.*
- Blazor: Classic Server mode — NO @rendermode InteractiveServer on pages
- Format specifiers in Razor: use .ToString("P0"), not @(value:P0)

--------------------------------------------------------------------------------
KEY TECHNICAL NOTES
--------------------------------------------------------------------------------

- ConfigFieldDefinition positional ctor order:
    (Key, Label, Description, FieldType, IsRequired, DefaultValue?, SelectOptions?)
- ValidateConfig(string configJson) — takes string, not AgentConnectionConfig
- JudgeVerdict is a record (supports "with {}" copy expression)
- API key middleware: runs after standard auth, SHA256 hash lookup in ApiKeys table
- Run execution: creates Run entity + enqueues TestRunJob to IMessageQueue "test-runs" queue
- CSV export: uses window.downloadCsvFromBase64 JS interop (defined in App.razor)
- EF tools startup project: src\Core\mate.Data (not WebUI — needs Design package)
- _Imports.razor includes @using mate.Data and @using mate.Domain globally
- Switch expressions with "< value =>" in Razor @code can cause parser issues — use if-else
